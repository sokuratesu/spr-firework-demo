New — Yuki's Question

Rotation Behavior: Same Hashtags, Multiple Videos

Question from Yuki: If multiple videos share the same hashtags and we embed a StoryBlock with max_videos="1", will it always show the same video?

Answer (confirmed by Spencer): StoryBlock auto-rotates sequentially. When multiple videos match the same hashtag filter, the embed cycles through them in order — Video C first, then Video D on the next display, then back to C. It does NOT always show the same video.

Below: a live StoryBlock filtering for (and he tv bravia-3 video setup) with max_videos="1". Two videos match (C & D). Click the reload button to destroy and re-create the embed — you should see it cycle between the two setup videos.

/* Two videos match this filter: Video C and Video D */
<fw-storyblock channel="sony_electronics_us"
    hashtag="(and he tv bravia-3 video setup)"
    max_videos="1" ></fw-storyblock>
Reloaded: 0 times
Live StoryBlock — sequential rotation (2 matching videos)
What to look for: Each time you reload, the embed should show a different one of the two 0:59 setup videos. If you see the same video twice in a row, try a third reload — the rotation is sequential, so it depends on where in the cycle you are.
New — Option Comparison

Pinned vs Rotating: Thank You Page Options

For the Thank You page, we have two approaches. Both are shown live below so Yuki and Tomo can compare.

Option A — Rotation

Let it rotate (recommended)

Keep the broad hashtag filter. Each time a customer finishes registration, they see a different setup video. Fresh content on repeat visits.

<fw-storyblock
  hashtag="(and he tv bravia-3 video setup)"
  max_videos="1" >
Option A: Rotating StoryBlock
Pros: Content stays fresh. No need to update embed code when new setup videos are added — they automatically enter the rotation.
Cons: Can't guarantee which specific video a customer sees first.
Option B — Pinned

Pin a specific video

Narrow the hashtag filter by adding D4 (SKU) or D6 (content type) to target a single video. Or use a Firework content ID to pin exactly one.

/* Add more hashtags to narrow to 1 match */
<fw-storyblock
  hashtag="(and he tv bravia-3 video tutorial)"
  max_videos="1" >
Option B: Pinned StoryBlock (only 1 tutorial video matches)
Pros: Full control over which video shows. Consistent experience.
Cons: Need to update the hashtag filter (or content ID) if you want to change the video. New videos don't appear automatically.
