Updated — Yuki's Question

Rotation Behavior: Same Hashtags, Multiple Videos

Question from Yuki: If multiple videos share the same hashtags and we embed a StoryBlock with max_videos="1", will it always show the same video?

Answer (confirmed via Firework docs + testing): Yes — with max_videos="1", StoryBlock always shows the most recently uploaded matching video, and it stays the same across page loads. The "sequential rotation" only happens within a single viewing session after the current video finishes playing, not between page loads.

What this means

Scenario Behavior
max_videos="1" Always shows the same (most recent) video on every page load. No rotation between visits.
max_videos="3" Loads 3 videos. Starts with the most recent, then auto-advances to the next after each finishes playing.
Page reload / new visit Resets to the same starting video. No built-in randomization across page loads.
sort_order="random" Does not exist. There is no built-in attribute for randomizing video order on load.

Given the rotation findings, here are three approaches for the Thank You page. All shown live below for comparison.

Option A — Auto-Advance Playlist

Show multiple videos, auto-advance after each finishes

Use max_videos="3" or higher. Customer sees the first setup video, and if they keep watching, it plays through the others automatically. Simple, no custom code needed.

<fw-storyblock channel="sony_electronics_us"
    hashtag="(and he tv bravia-3 video setup)"
    max_videos="3" ></fw-storyblock>
Option A: Auto-advance (max_videos="3")

Option B — Pinned Single Video

Always show one specific video

Narrow the hashtag filter to match only one video (e.g., add D6=tutorial to target the 4:24 video). Full control, always predictable.

<fw-storyblock channel="sony_electronics_us"
    hashtag="(and he tv bravia-3 video tutorial)"
    max_videos="1" ></fw-storyblock>
Option B: Pinned (only 1 tutorial video matches)
Pros: Full control. Consistent, predictable experience.
Cons: Manual update needed to change the video. No variety for repeat visitors.
Option C — JS Randomization (Recommended)

Randomize on each page load via JavaScript

React picks a random content type (setup, feature, tutorial) on each page load and passes it to the StoryBlock. Different customers see different videos. No taxonomy hacks needed — the randomization lives in the application logic.

JavaScript Randomization — How It Works

This section shows the complete implementation pattern for React. The approach is simple: on page load, pick a random content type from the available options, build the hashtag expression, and pass it to the Firework embed. The taxonomy stays clean — no duplicate videos, no fake hashtags.

// === React implementation pattern ===

// Content types available for the registered product
const contentTypes = ['setup', 'feature', 'tutorial'];

// Pick a random one on each page load
const randomType = contentTypes[Math.floor(Math.random() * contentTypes.length)];

// Build hashtag expression from registered product + random type
// d1, d2, d3 come from the user's registered product data
const hashtag = `(and ${d1} ${d2} ${d3} video ${randomType})`;

// Pass to Firework embed
<fw-storyblock
    channel="sony_electronics_us"
    hashtag={hashtag}
    max_videos="1"
/>
Why this approach is clean: The randomization happens in the application layer (React), not in Firework's taxonomy. The six-dimension hashtag structure stays exactly as agreed. We're simply varying which D6 content type we ask Firework to filter by on each page load. If Sony adds new setup or feature videos later, they automatically enter the rotation — no code changes needed.

Alternative: Randomize at D5 level (shorts vs videos)

Instead of randomizing D6 (content type), we could randomize D5 (format) — sometimes showing a short clip, sometimes a longer video. Or combine both for maximum variety. The pattern is the same, just different dimensions.
