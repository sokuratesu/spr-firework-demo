Thank You Page: Guest vs Logged-in Routing

The video embed is the same for both user types. The difference is where the "Explore Video Library" button takes them. Toggle between guest and logged-in to see the routing change.

View as:
Guest user
"Explore Video Library" button → Login / Sign up page
Guest flow: The button takes the user to the login or sign-up page. After creating an account or logging in, they can access the full Video Hub. The same video embed plays regardless of login status.
/* React checks login status and routes accordingly */

const destination = isLoggedIn
  ? '/video-hub'
  : '/login?redirect=/video-hub';

/* The Firework embed is identical for both */

    max_videos="1"
    pip="true" /* enables auto-float on scroll */ />

Floating Player Behavior (Picture-in-Picture)

When a user plays a video and then scrolls past it, the video automatically minimizes into a floating player in the bottom corner. This is enabled by adding pip="true" to the StoryBlock embed. The video must be playing for this to trigger.

1
Click any video above to start playing it.
2
Scroll down past the video while it's playing. It automatically shrinks to a floating widget in the corner.
3
The user can continue browsing the page while the video plays. Click the floating player to expand it again, or X to close.
pip="true" enables this. Add it to any fw-storyblock embed. Note: this only works on fw-storyblock, not on fw-embed-feed (Player Deck, Grid, Carousel, etc.). The video must be playing for the auto-float to trigger.
